Решение ошибки Cursor

"Too many free trial accounts are in use on this machine"

Выберите вашу операционную систему

1 Откройте PowerShell от имени администратора
2 Нажмите Win + X и выберите "Windows PowerShell (Администратор)" или "Терминал (Администратор)"
3 Скопируйте и выполните команду ниже
irm https://raw.githubusercontent.com/yuaotian/go-cursor-help/refs/heads/master/scripts/run/cursor_win_id_modifier.ps1 | iex
📝 Примечание: Скрипт автоматически определит PowerShell 7 и запросит права администратора через UAC. Если PowerShell 7 не установлен, будет использован Windows PowerShell.
4 Перезапустите Cursor, чтобы изменения вступили в силу
1 Откройте приложение Terminal (Терминал)
2 Скопируйте и выполните команду ниже
curl -fsSL https://raw.githubusercontent.com/yuaotian/go-cursor-help/refs/heads/master/scripts/run/cursor_mac_id_modifier.sh -o ./cursor_mac_id_modifier.sh && sudo bash ./cursor_mac_id_modifier.sh && rm ./cursor_mac_id_modifier.sh
🔐 Примечание: Команда sudo потребует ввода пароля администратора. При вводе пароль не отображается на экране — это нормально.
3 Перезапустите Cursor, чтобы изменения вступили в силу
1 Откройте терминал
2 Скопируйте и выполните команду ниже
curl -fsSL https://raw.githubusercontent.com/yuaotian/go-cursor-help/refs/heads/master/scripts/run/cursor_linux_id_modifier.sh | sudo bash
⚠️ Важно для Linux: Скрипт ищет установку Cursor в стандартных путях (/usr/bin, /usr/local/bin, $HOME/.local/bin, /opt/cursor, /snap/bin). Убедитесь, что Cursor установлен в одном из этих расположений.
3 Перезапустите Cursor, чтобы изменения вступили в силу

Что делает этот инструмент?

Программа изменяет файл конфигурации Cursor storage.json, расположенный по следующим путям:

  • Windows: %APPDATA%\Cursor\User\globalStorage\storage.json
  • macOS: ~/Library/Application Support/Cursor/User/globalStorage/storage.json
  • Linux: ~/.config/Cursor/User/globalStorage/storage.json

Инструмент генерирует новые уникальные идентификаторы для полей:

  • telemetry.machineId
  • telemetry.macMachineId
  • telemetry.devDeviceId
  • telemetry.sqmId
